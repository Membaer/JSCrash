Aufruf von Javascript Befehlen von oben nach unten --> Reihenfolge beim Aufruf beachten
Wenn eine Variable erst unterhalb der Funktion (alert) initialisiert wird, dann wird die Funktion übersprungen
Es gibt nur vier Datentypen in JS - numbers, text, boolean, objects
Variablen können überall (ohne Deklaration) intialisiert und befüllt werden
Es gibt Kontrollstrukturen in JS - Entscheidungen, Schleifen und Sprunganweisungen
Entscheidungen prüfen eine Kondition und führen je nach Entscheid einen Block aus (if, else)
Schleifen prüfen eine Kondition und führen einen Block aus bis diese Kondition erfüllt ist (for(i=0,i<=10,i++){};)
Sprunganweisungen können aus Funktionen, Schleifen etc. herausspringen
Funktionen, dienen dazu JS bewusst / gezielt aufzurufen / zu starten (nicht einfach beim Laden der Seite)
Funktionsbeschreibungen können mit "Platzhaltern" ergänzt werden, die erst beim Funktionsaufruf befüllt werden
Funktionen können sich selbst aufrufen
Funktionen können durch Eventhandler aufgerufen werden (onclick, onmouseout...)
Objekte können per Arraynotation links[0] (erster Link) oder per Punktnotation links.mylink aufgerufen werden
DOM - Objekte können per Namen, TagNamen etc. aufgerufen werden, bei Aufruf aller Bilder kann über Arraynotation das gewünschte Bild geholt werden
DOM: document.images[2].href --> ändere den href vom dritten Bild
DOM - Element können alle HTML Eigenschaften nutzen z.B. ("img").width / .height ... und z.B. innerHTML für Texte
Um HTML Gerüst von Fkt. (onmouseover) freizuhalten, können anstelle von HTML Event-Handlern auch dieselben DOM Event Handler genutzt werden
DOM Event Handler dürfen nicht zu früh geladen werden, sonst könnte das DOM noch nicht komplett aufgebaut sein
Besser ist die anonmyen Funktionen bei window.onload mit zu laden
Funktionsreferenzen window.onload = init; - immer ohne Klammern
Statt css, kann JS über das Style attribut die WErte setzen, aber namen ändern - js: backgroundColor vs css: background-color
Bei einem Loop (for(){}) über elementsbytagname kann man den array index mit .length ansprechen
AJAX kann Element vom SErver über Javascript nachfragen - dazu get anfrage an URL (Server) und Anzeige in HTML
